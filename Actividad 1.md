# EQUIPO 1: Actividad 1: Análisis de Proyecto Fallido - Quibi

## 1. Análisis del Fracaso: Quibi
Quibi nació con una inversión de 1.75 billones de dólares y cerró en apenas 6 meses. A continuación, se detallan las causas del fallo:

### A. Problemas de Planificación
* *Modelo de negocio rígido:* Se centró exclusivamente en dispositivos móviles en un momento donde la pandemia obligó a las personas a consumir contenido en televisores (Smart TVs).
* *Restricciones de producto:* Inicialmente, la app bloqueaba las capturas de pantalla, impidiendo que el contenido se volviera viral en redes sociales.
* *Muro de pago temprano:* No ofrecieron una versión gratuita con anuncios (AVOD), lo que limitó la adquisición de usuarios frente a competidores como TikTok o YouTube.

### B. Problemas de Comunicación e Integración
* *Conflicto de visión:* Existió una brecha cultural entre la directiva de Hollywood (enfocada en contenido de alto presupuesto) y la visión tecnológica de Silicon Valley.
* *Falta de integración de mercado:* No se escuchó la demanda del usuario por contenido compartible y multiplataforma.

### C. Integración de Código o Proyecto
* *Exceso de ingeniería:* Se invirtió demasiado esfuerzo en la tecnología Turnstyle (giro de pantalla) que complicaba la producción, en lugar de priorizar la compatibilidad con otros ecosistemas (Web, TV).

## 2. Estrategia Kanban para la Reestructuración

Para rescatar el proyecto, implementaremos un flujo de trabajo ágil basado en *Kanban* para permitir entregas rápidas y adaptabilidad.

### Roles del Equipo
| Rol | Responsabilidad |
| :--- | :--- |
| *Product Owner* | Priorizar el backlog basándose en el feedback real del usuario. |
| *Kanban Master* | Eliminar cuellos de botella y asegurar que el equipo siga el flujo. |
| *Fullstack Developer* | Desarrollo de la versión Web y compatibilidad con Smart TV. |
| *UX/UI Designer* | Adaptar la interfaz para que sea socialmente compartible. |
| *QA Engineer* | Pruebas de rendimiento en múltiples dispositivos y resoluciones. |

### Backlog Inicial (Tareas prioritarias)
1. [ ] *Habilitar compatibilidad con Smart TVs:* (LG, Samsung, Apple TV).
2. [ ] *Eliminar bloqueo de capturas:* Permitir la creación de clips y GIFs para redes sociales.
3. [ ] *Lanzamiento de Plan Gratuito:* Implementar sistema de anuncios para atraer usuarios.
4. [ ] *Versión Web:* Permitir la reproducción de contenido desde cualquier navegador.
5. [ ] *Sistema de Recomendación:* Integrar algoritmos de IA para personalización.
6. [ ] *Integrar funciones de comunidad:* Comentarios y perfiles de usuario para fomentar la interaccion

### Objetivos por Sprint (Hitos)
* *Iteración 1:* "Viralidad y Apertura" - Foco en funciones sociales y fin de la exclusividad móvil.
* *Iteración 2:* "Multiplataforma" - Lanzamiento de versiones para Web y TV.
* *Iteración 3:* "Monetización Híbrida" - Implementación del modelo gratuito con publicidad.
